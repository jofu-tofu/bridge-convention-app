# Research Question

## Core Question
What are the best SEO principles and technical practices for maximizing search engine visibility of a niche educational single-page application (SPA) built with WASM, specifically a bridge card game practice app?

## Key Concepts
- **SEO for SPAs**: single-page application SEO, JavaScript SEO, client-side rendering SEO, crawlability
- **Technical SEO**: site structure, meta tags, semantic HTML, structured data, page speed, Core Web Vitals, mobile-friendliness
- **WASM/JavaScript rendering**: WebAssembly SEO implications, rendering strategies (SSR, SSG, prerendering), Googlebot rendering
- **Niche educational content strategy**: long-tail keywords, topical authority, content hubs, educational content SEO
- **Discoverability for web apps**: app indexing, progressive web apps SEO, deep linking

## Native Field
Search Engine Optimization (SEO) / Web Development

## Adjacent Fields
- **Information Retrieval**: how search engines rank and index content, particularly dynamic/JS-heavy content
- **Educational Technology**: how niche educational tools gain organic visibility, content marketing for learning platforms
- **Human-Computer Interaction**: user engagement signals that affect SEO (dwell time, bounce rate, interaction patterns)

## Local Context Found
- The app is a Svelte 5 + TypeScript SPA with WASM (Rust) backend logic
- Deployed via Docker/Caddy on a VPS
- Currently no SSR or prerendering — all content renders client-side via WASM
- The app teaches bridge bidding conventions (niche educational topic)
- Has a free/paid tier split (learn free, practice gated)
- No existing SEO strategy or content pages identified in the codebase

## Relevance Filter
Prune findings that focus on:
- E-commerce SEO (product pages, shopping feeds)
- Large-scale news/media SEO
- Local SEO / Google Maps optimization
- Social media marketing (unless directly tied to search visibility)
- Paid advertising / SEM

Prioritize findings relevant to:
- Small niche educational web apps
- SPAs with limited server-rendered content
- WASM-based applications
- Content strategy for specialized knowledge domains (card games, hobbies)
