## Vocabulary Discovered

- **Contrasting cases** (Schwartz/Bransford): juxtaposed examples varying the deep feature while holding surface constant — the pedagogical load-bearing signal of peer-bid conventions.
- **Alignable differences** (Gentner / Kurtz): differences that fall on a shared dimension; the *teachable* differences that emerge under juxtaposition. Non-alignable differences are noise.
- **Structural alignment / structure-mapping / systematicity:** relational (not surface) features get encoded when multiple exemplars are aligned.
- **Analogical encoding:** learning by comparing two partly-understood examples; ~3× transfer vs sequential study of the same cases.
- **High-variability worked examples / variability effect:** co-presenting surface-different instances of a shared schema beats low-variability sets for transfer.
- **Progressive alignment:** scaffolded comparison from high-similarity to increasingly dissimilar pairs; supports ordering of peer examples on a reference page.
- **Procedural flexibility** (Rittle-Johnson): ability to select among multiple correct methods — direct analogue for partnership choosing among peer bids.
- **Expertise reversal effect:** worked-example / juxtaposition benefits invert for experts — relevant to whether layered presentation (collapse for experts, expand for novices) is warranted.
- **Preparation for future learning (PFL):** contrasting cases prepare learners for later "telling"; relevant to whether reference pages should sit before or alongside practice.
- **Template theory** (Gobet/Simon): expert knowledge as parameterized templates with variable slots — structural analogue for convention schema with instance slots (peer bids).
- **Monochrestic knowledge** (Gobet): single-use memorized sequences; contrasts with pattern-family knowledge. Hero-bid reference = monochrestic; peer-family reference = pattern-family.
- **Discriminator** (OpenAPI): explicit field naming the axis of variation among peer schema variants. Directly portable to "what differentiates peer bids" (suit shown, strength class, trigger context).
- **Functional family** (music theory): equivalence class of chords sharing harmonic role; parallels peer-bid convention family.
- **Transposition** (chess): alternative move-order reaching the same structural position; the pedagogical anti-pattern when taught as "deviation from main line."
- **Typicality / frequency-induced prototypicality:** exemplar-theory mechanism by which repeated prominence of one instance makes peers feel like lesser members — mechanism by which a hero bid erodes peer legitimacy.
- **Peer entry-point / family of calls:** project-native terms; ACBL usage closest match ("related agreements").

## Landscape Assessment

**Major debates / tensions:**
- **Expertise reversal effect** vs **variability / contrasting-cases benefit.** Novice benefit is robust; expert learners can be harmed by redundant juxtaposition (CLT retrospective). Implies reference pages may need audience-adaptive layering (expanded peer comparison for novices, collapsed canonical listing for experienced players).
- **Explanation-level granularity** (Wittwer & Renkl). Principle-level explanations beat surface-level; implies the *shared schema* needs its own authored explanation, and per-peer-bid glosses should not simply restate surface features.
- **Within bridge practice: peer-parallel layouts are already dominant** where the convention is structurally peer-bid (DONT, Bergen, Two-way NMF, Unusual NT, ACBL convention card). Hero-plus-branches is a minority format tied to conventions where one bid genuinely dominates (Puppet Stayman, Stayman). **The project's current schema is the minority format for the majority of multi-bid conventions.**

**Apparent gaps:**
- No bridge-pedagogy peer-reviewed literature; all bridge evidence is authoritative-practice only.
- No direct empirical study comparing hero vs peer presentation in any reference-page domain (bridge, chess, API, grammar). Recommendation transfers by structural analogy.
- No NN/g or design-handbook synthesis on parallel reference families at reference-page granularity; documentation handbooks address parallel construction only at sentence/list level.
- Limited treatment of *how many* peer instances cross the threshold where co-presentation helps vs overwhelms novices (Paas variability research mostly 2–4 instances).
- Chess pedagogy touches transpositions but no empirical study formally compares main-line-first vs pattern-first opening reference design.
