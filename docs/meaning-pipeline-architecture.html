Convention Protocol & Meaning Pipeline

Architecture review — how bids are selected, graded, and explained. Emphasis on assumptions and robustness boundaries.

System Overview

The system determines the correct bid for a given hand and auction position, then grades the user's bid against it. Every convention is authored as a ConventionBundle — a self-contained package of declarative meaning surfaces, a finite state machine, fact extensions, and teaching metadata. A convention-universal pipeline evaluates all bundles through the same 5-step transformation.

What a MeaningSurface Looks Like

meaningId: "stayman-2C"
semanticClassId: "stayman.ask"
surfaceGroupId: "stayman-responses"
clauses: [
{ factId: "hand.hcp", operator: "gte", value: 8 },
{ factId: "bridge.hasFourCardMajor", operator: "boolean", value: true }
]
encoding: { defaultCall: { type: "bid", level: 2, strain: "C" } }
ranking: { recommendationBand: "should", specificity: 2, modulePrecedence: 0 }

Each surface is a declarative rule: "if these fact conditions hold, this bid means X." No imperative logic.

Conversation Machine (FSM)

Each bundle has a ConversationMachine — an SCXML-inspired hierarchical FSM that tracks where in the bidding conversation we are. The machine replays the entire auction from idle, matching each bid against transitions. The final state determines which surfaceGroupIds are active.

  direction LR
  [*] --> idle
  idle --> r1_response : partner opens 1NT
  r1_response --> r2_stayman : self bids 2C
  r1_response --> r2_transfer : self bids 2D/2H
  r2_stayman --> r3_stayman_rebid : partner responds
  r2_transfer --> r2_transfer_accept : partner completes
  r2_stayman --> r2_opponent_action : opponent interferes
  r1_response --> terminal : self passes

Key FSM Features

Hierarchical States

Child states inherit parent transitions. DONT uses 21-state hierarchy with parent/child grouping for competitive/defensive phases.

Submachines

States can invoke a named submachine (push), execute it, then return (pop). Smolen uses this for guard-based routing in NT.

Loop Counters

loopConfig.maxIterations prevents infinite cycles. Auto-exits to exitTarget when reached.

Machine Registers

Transitions carry MachineEffects that set forcingState, obligation, agreedStrain, captain, and systemCapabilities.

Transition Matching

6 match kinds: call, pass, any-bid, opponent-action (with level/strain filter), predicate, submachine-return.

Guards

Transitions can have a guard(snapshot) function that checks register values before allowing the transition.

The 5-Step Pipeline

The core pipeline is a pure transformation in meaning-strategy.ts. No caching, no side effects, no state mutation. Called once per bid decision.

flowchart TD
  A["1. SELECT: Machine produces activeSurfaceGroupIds\nFilter MeaningSurfaces to active groups"] --> B
  B["2. COMPOSE: Apply CandidateTransforms\n(suppress / inject / remap surfaces)"] --> C
  C["3. FACTS: evaluateFacts(hand, evaluation, catalog)\n3-tier: primitive, bridge-derived, module-derived\n+ optional relational + posterior"] --> D
  D["4. EVALUATE: evaluateAllSurfaces(surfaces, facts)\nEach clause checked against evaluated facts\nResult: MeaningProposal per surface"] --> E
  E["5. ARBITRATE: evaluateProposal through 4 gates\nClassify into truth/acceptable sets\nRank by band, specificity, precedence\nSelect winner"]

The 4-Gate Sequence

Every proposal passes through 4 gates in fixed order. First failure eliminates the candidate.

  G4 --> OK["Encoded\nProposal"]

GateWhat It ChecksCurrent Status
Semantic Applicability All clauses satisfied? (clauses.every(c => c.satisfied)) Active

Encoder Availability Can the meaning be encoded as a legal call? Pass-through — subsumed by legality

Gates 2 and 3 always pass today. They exist as placeholders for future enforcement (forcing obligations, complex encoder resolution).

Arbitration

After gate checks, surviving proposals are classified into truth set (all clauses satisfied + legal) and acceptable set (clauses failed but legal, band "may" or better). The truth set is sorted and the winner selected.

flowchart TD
  EP["Encoded Proposals"] --> CL["classifyIntoSets()"]
  CL --> TS["Truth Set\n(satisfied + legal)"]
  CL --> AS["Acceptable Set\n(unsatisfied + legal)"]
  TS --> SORT["Sort by ranking:\n1. recommendationBand\n2. specificity (higher wins)\n3. modulePrecedence\n4. intraModuleOrder"]
  SORT --> DEDUP["Deduplicate by\nsemanticClassId aliases"]
  DEDUP --> WIN["Winner = first in\ndeduplicated list"]
  WIN --> RES["ArbitrationResult"]
  AS --> RES

Ranking Hierarchy

BandPriority ValueMeaning
must0 (highest)Obligatory — forced bid
should1Preferred conventional bid
may2Correct but not preferred
avoid3 (lowest)Fallback; correct but inadvisable

Ties in band are broken by specificity (more specific wins), then modulePrecedence (lower wins), then intraModuleOrder.

Teaching Resolution

The teaching layer grades the user's bid against the pipeline's answer. Five grades, evaluated in cascade order:

Correct

Exact match with primary recommendation

CorrectNotPreferred

In truth set but not the top pick

Acceptable

In alternative groups or has preferred/alternative priority

NearMiss

Same intent family, fails a constraint

Incorrect

Not in any recognized set

Resolution Process

  1. Priority filter: Eligible candidates with preferred or alternative priority become acceptable bids
  2. AlternativeGroup lookup: Cross-convention alternatives (bypasses tree-path exclusivity)
  3. IntentFamily-aware credit: equivalent_encoding relations get full credit
  4. Deduplication: Higher-credit version wins when a bid appears in both sources
  5. Near-miss detection: Candidates sharing an intent family with the matched bid that have failed conditions

Key Assumptions

The pipeline's correctness depends on these assumptions. Each is annotated with robustness status.

Robust Deterministic grading

Same hand + same auction = same grade. No randomness in the pipeline. Posterior enrichment is a sidecar and does not affect the truth set in V1.

Robust Convention-universal core

All pipeline stages (fact eval, clause eval, arbitration, encoding) are generic. Convention-specific logic lives only in definitions/. Validated by 4 distinct convention patterns.

Robust Fact evaluation is topologically sorted

Facts declare derivesFrom dependencies. The evaluator sorts them before execution. Cycle detection is implicit (visited set).

Moderate FSM replays entire auction

The machine replays all auction entries from idle on every call to suggest(). O(n) per bid where n = auction length. Fine for bridge (max ~40 bids) but fundamentally non-incremental. Caching by auction length mitigates repeated calls at the same position.

Moderate Single-winner arbitration

The pipeline selects exactly one winner from the truth set (first after sorting). When multiple surfaces tie on all ranking dimensions, the winner is determined by array order. This is intentional (surfaces are authored in precedence order) but implicit.

Moderate Fail-closed on missing facts

If a fact is missing from the catalog, the clause is marked unsatisfied. Correct for safety, but a typo in a factId silently eliminates a surface with no diagnostic warning.

Moderate Obligation gate is a no-op

Gate 2 (obligation-satisfaction) always passes. Forcing obligations are handled separately at the strategy-chain level via createForcingFilter. This split means the pipeline's elimination trace doesn't show obligation failures.

Fragile No FSM validation at authoring time

machine-validation.ts exists (orphan/dangling ref checks) but is not automatically run when bundles are registered. A malformed machine is only caught at runtime when no transition matches.

Fragile surfaceGroupId coupling

The machine produces activeSurfaceGroupIds by string. These must exactly match the surfaceGroupId on MeaningSurfaces. A typo in either place silently produces an empty surface set — the pipeline returns null (no bid) with no diagnostic.

Fragile Predicate transitions are opaque

predicate match kind takes an arbitrary (call, seat, snapshot) => boolean. These are not serializable, not testable in isolation from the FSM, and invisible to validation tooling.

Moderate Teaching relies on resolvedCandidates

Teaching resolution builds from BidResult.resolvedCandidates mapped from truth + acceptable sets. If the arbitrator drops a candidate (e.g., semantic class dedup), it vanishes from teaching — the student never learns about the eliminated alternative.

Moderate Relational facts require publicCommitments

Facts like bridge.fitWithBoundSuit check partner's minimum suit length from publicCommitments. If the commitment extractor misses a bid (e.g., opponent interference disrupts the machine), the fit check defaults to 0.

Known Gaps & Open Questions

AreaGapImpact
Spec Posterior semantic model not finalized. Joint distribution variables, LatentWorld representation, and fact catalog posterior compilers are unresolved. Blocks WitnessSpecIR wiring, inference spectrum (difficulty config), and probabilistic fact enrichment beyond stub.
Spec Evidence group correlation model not formalized as types. Four target group kinds identified but no IR. Posterior combiner accuracy depends on correct within-group semantics.
Coverage Only 2 of 6 convention patterns exercised: Pattern 1 (fixed-sequence ask-and-tell) and Pattern 3 (competitive/defensive). Submachines exercised (Smolen). Patterns 2, 4-6 not yet tested. Assumptions about convention-universality are only validated against existing patterns.
Coverage Host-attachment activation (module activating from another module's exported capabilities) is spec-designed with vocabulary resolved but not exercised by a convention. Negative Doubles and Fourth Suit Forcing are blocked until a convention exercises this pattern.
Migration Pipeline still consumes MeaningSurface[], not DecisionSurfaceIR[]. Adapter exists in test code. Two-phase evaluation runtime wired but not the primary consumption path. Dual code paths increase maintenance burden. DecisionSurfaceIR migration would unify them.
Resilience No static analysis for surface/machine consistency. surfaceGroupId typos, missing fact catalog entries, and unreachable machine states are caught only by tests or runtime nulls. Authoring errors can silently produce wrong answers. Mitigated by behavioral tests but not by tooling.
Resilience WitnessSpecIR types exist but are not wired to deal generation. Raw DealConstraints used instead. Deal generation can produce hands that are impossible for the convention, leading to "no bid found" results.

Verdict Summary

Strong
  • Deterministic, pure pipeline
  • Convention-universal core infrastructure
  • Declarative surface authoring
  • 5-grade teaching resolution with intent families
  • Rich provenance & evidence trail
Watch
  • 2 of 4 gates are no-ops
  • String-based coupling (surfaceGroupId, factId)
  • FSM replays full auction each time
  • Implicit tie-breaking by array order
  • Posterior enrichment is stub-only
Risk
  • No static validation tooling for bundles
  • Only 2/6 convention patterns exercised
  • Predicate transitions are opaque black boxes
  • Posterior & WitnessSpec design incomplete
